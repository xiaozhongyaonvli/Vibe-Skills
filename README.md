# VibeSkills

> 把分散的 skills、插件、工作流和经验，收进一个更容易调用、更不容易失控的 AI agent 系统。

`VibeSkills` 是对外名字，`VCO` 是它背后的受管运行时。

它不是另一个 skills 列表。
它更像一个瑞士军刀式的智能整合 agent 系统：帮你把优秀但分散的 skills、项目、插件和实践经验收进同一套可调用、可治理、可留痕的工作流里。

## 为什么会有它

我一直在看很多很好的工具、skills、插件和 agent 项目。

它们各有所长，也都很有价值。但真实使用时，问题很快就出来了：找起来麻烦，互相之间容易冲突，不知道当前场景该用哪个，也不知道什么时候该先澄清需求、什么时候该先验证、什么时候该保留留痕、什么时候该停下来。

于是这件事慢慢变成了一个整合工程。
我开始把踩过的坑、整理过的规范、验证流程、安全边界、长期维护经验，还有各类优秀项目的长处，一点点收进同一个系统里。

我想做的不是一个“很厉害的工具箱”。
我想做的是一套更省心的 AI 工作方式。

## 它解决什么问题

如果你也在重度使用 AI，你大概率会遇到这些问题：

- skills 太多，不知道现在该用哪个
- 项目、插件、工作流彼此有重叠，也彼此有冲突
- AI 很容易跳步骤，没澄清需求就直接开做
- 做完之后没有验证、没有留痕，长期维护很容易黑盒化
- 安装和接入门槛高，普通用户很难舒服地开始

`VibeSkills` 想做的，就是把这些选择成本、组合成本和失控成本降下来。

## 它是怎么工作的

你可以把它理解成三层：

### 1. 智能路由

在合适的场景下，AI 不需要你显式记住“这次该调用哪个 skill”。

`VibeSkills` 会把逻辑路由和 AI 智能路由结合起来，尽量把合适的能力放到合适的场景里，让调用更自然，而不是靠你手动背技能表。

### 2. 受管工作流

它不只是在“调工具”。
它更关心工作怎么做才稳定。

所以这套系统会尽量把需求澄清、确认、执行、验证、回顾、留痕这些步骤收进统一流程里，避免 AI 一上来就黑盒式开跑。

### 3. 整合能力

这里不只有 skills。

还有插件、项目、工作流设计、AI 规范、安全边界、长期维护经验，以及我自己在实践里踩过的坑。
`VCO` 负责把这些能力组织成一个更统一的运行时，而不是让它们继续散落在不同角落里。

## 它适合谁

`VibeSkills` 主要适合这几类人：

- 想让 AI 更稳定地帮自己做事的普通用户
- 已经在重度使用 AI / Agent / 自动化的进阶用户
- 想把 AI 工作流做得更规范、更可维护的个人或小团队
- 已经厌倦“技能太多但不好用”的人

如果你只是想找一个单点工具，这个仓库可能不是最轻的选择。
如果你想把 AI 用得更稳、更顺、更长期，它会更有意义。

## 为什么和普通 skills 仓库不一样

很多仓库解决的是“这里有什么”。
`VibeSkills` 更关心的是：

- 现在应该用什么，而不是展示有什么
- 先做什么，后做什么，而不是让 AI 一股脑往前冲
- 哪些能力可以组合，哪些地方需要边界
- 怎么验证结果，怎么留下证据，怎么避免长期黑盒化

它不是把东西堆得更多。
它是想让这些东西更好用。
