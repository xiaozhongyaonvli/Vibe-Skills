# OpenClaw 文档版式收口需求文档

**日期**: 2026-03-25
**任务类型**: 文档治理 - OpenClaw 文案与版式收口
**优先级**: 中
**执行模式**: 小范围文案修正

---

## 🎯 目标（Goal）

调整 OpenClaw 在公开文档中的表达方式，避免被单独拉出来强调，保持 README 与安装文档的版式和语气正常、并列、自然。

## 📦 交付物（Deliverable）

1. README / README.zh 中更自然的 OpenClaw 文案
2. OpenClaw 说明文档标题与入口文案的收口版本
3. 一轮只包含版式与文案修正的提交

## 🔒 约束条件（Constraints）

1. 不改动真实支持范围
2. 不改动 OpenClaw 的安装口径
3. 不扩展新功能
4. 只收口文案与版式，不重写整套安装文档

## ✅ 验收标准（Acceptance Criteria）

1. README 中不再出现带特殊强调感的 OpenClaw 单独入口文案
2. OpenClaw 说明页标题不再使用“运行时核心预览路径”式表达
3. 页面仍保留正常可访问的 OpenClaw 说明入口
4. 文案语气与其他宿主保持一致

## 🚫 非目标（Non-Goals）

1. 不删除 OpenClaw 支持说明
2. 不修改脚本、适配器或测试
3. 不调整其他宿主的支持口径
