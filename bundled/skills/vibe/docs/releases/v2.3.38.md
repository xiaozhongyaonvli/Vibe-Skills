# VCO Release v2.3.38

- Date: 2026-03-14
- Commit(base): bb98ece

## Highlights

- Removed author-machine path leakage from the governed runtime config surfaces, so release-critical JSON no longer hard-codes a specific developer machine path.
- Switched the affected path-bearing configs to `${CODEX_HOME}` and `${VCO_EXTERNAL_ROOT}` contracts and updated the consuming governance/runtime scripts to resolve those tokens safely.
- Re-synced canonical, bundled, and nested-bundled mirrors and refreshed the smoke installed runtime proof so Linux-side testing can validate against the same governed release surface.
- Cut and aligned the public release surface to `2.3.38` across version governance, maintenance markers, changelog, release ledger, and mirrored release assets.

## Migration Notes

- If you consume `config/dependency-map.json`, `config/upstream-corpus-manifest.json`, or `config/ruc-nlpir-runtime.json` directly, resolve `${CODEX_HOME}` and `${VCO_EXTERNAL_ROOT}` through the shared governance helpers instead of assuming absolute host paths.
- Installed-runtime verification should be re-run after upgrading so `skills/vibe/outputs/runtime-freshness-receipt.json` reflects the new tokenized path contract.
- Linux operators can now test this release without inheriting the previous Windows author-machine path assumptions from the governed config layer.
