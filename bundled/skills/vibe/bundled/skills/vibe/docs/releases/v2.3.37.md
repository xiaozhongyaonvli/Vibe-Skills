# VCO Release v2.3.37

- Date: 2026-03-13
- Commit(base): 8fec70f

## Highlights

- Promoted `scrapling` into the default governed `full` install lane so the standard full-profile path now carries an explicit local scraping/runtime surface.
- Extended both install scripts to attempt `scrapling[ai]` provisioning during external-tool installation and kept the result honest by warning when the CLI still is not callable.
- Updated bootstrap doctor semantics to expose three different default surfaces instead of flattening them into one ambiguous "full-featured" story:
  - `scrapling` = default full-profile local runtime surface
  - `Cognee` = default long-term graph-memory enhancement lane
  - `Composio / Activepieces` = prewired but setup-required external action surfaces
- Reworked the install-facing docs so Windows and Linux operators can distinguish `minimum viable`, `recommended full-featured`, and enhancement/setup-required paths without overclaim.
- Recut the governed release surface to `2.3.37` and aligned release-facing metadata across governance source, changelog, release ledger, and release navigation.

## Migration Notes

- No runtime contract was widened beyond the install/documentation surface. `state_store` remains the session truth source, and `Cognee` remains an optional long-term enhancement rather than a replacement memory owner.
- `Composio` and `Activepieces` remain confirm-gated external action planes. This release makes them more visible and better documented; it does not silently auto-enable them.
- Host-managed plugins and plugin-backed MCP surfaces still require platform-side provisioning. A truthful first-run result may still be `manual_actions_pending`.
