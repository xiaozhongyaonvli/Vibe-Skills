# 安装文档去重需求文档

**日期**: 2026-03-23
**任务类型**: 文档治理 - 安装文档去重
**优先级**: 高
**执行模式**: 保持功能完整，消除重复

---

## 🎯 目标（Goal）

对安装文档进行去重优化，在**保持所有安装功能完整**的前提下，消除重复内容，提升文档的可维护性和可读性。

## 📦 交付物（Deliverable）

1. 重组后的安装文档结构
2. 提取的公共规则文档
3. 提取的配置说明文档
4. 精简后的4个安装提示词
5. 文档变更说明

## 🔒 约束条件（Constraints）

### 必须保持的功能
1. ✅ **所有安装选项** - 全量版本、框架版本的安装功能必须保留
2. ✅ **所有更新选项** - 全量版本、框架版本的更新功能必须保留
3. ✅ **所有配置说明** - VCO_AI_PROVIDER相关配置说明必须保留
4. ✅ **所有规则说明** - 13条安装规则必须保留
5. ✅ **所有宿主支持** - codex和claude-code的支持必须保留

### 必须优化的问题
1. ❌ **规则重复** - 13条规则在4个提示词中重复出现，需要提取
2. ❌ **配置重复** - VCO配置说明在4个提示词中重复出现，需要提取
3. ❌ **结构重复** - 4个提示词有70%的重复内容，需要模块化
4. ❌ **可维护性差** - 修改规则需要同步4处，需要集中管理

### 不允许的操作
1. ❌ 删除任何安装功能
2. ❌ 删除任何配置选项
3. ❌ 简化安装流程
4. ❌ 改变提示词的核心逻辑

---

## ✅ 验收标准（Acceptance Criteria）

### 去重标准
1. ✅ 公共规则提取到独立文档，4个提示词引用
2. ✅ 配置说明提取到独立文档，4个提示词引用
3. ✅ 提示词重复率从70% → <20%
4. ✅ 修改规则只需更新1处，不需要同步4处

### 功能完整性标准
1. ✅ 所有安装选项保持不变
2. ✅ 所有更新选项保持不变
3. ✅ 所有配置说明保持不变
4. ✅ 所有规则说明保持不变
5. ✅ 用户体验保持不变

### 可维护性标准
1. ✅ 文档结构清晰，易于导航
2. ✅ 公共内容集中管理
3. ✅ 提示词简洁，易于理解
4. ✅ 修改成本降低

---

## 🚫 非目标（Non-Goals）

1. ❌ 不改变安装流程的核心逻辑
2. ❌ 不删减任何安装功能
3. ❌ 不改变提示词的交互方式
4. ❌ 不修改安装脚本本身

---

## 🤔 假设（Assumptions）

1. 用户希望安装文档更易于维护
2. 提取公共内容不会影响用户体验
3. 提示词可以通过引用外部文档来减少重复
4. 当前的安装功能和流程是正确的

---

## 📋 依赖（Dependencies）

1. 需要访问 `docs/install/one-click-install-release-copy.md` 文件
2. 需要理解当前安装文档的完整内容和结构
3. 需要保持与其他文档的链接一致性
