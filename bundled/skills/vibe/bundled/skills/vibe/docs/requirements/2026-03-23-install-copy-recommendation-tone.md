# 安装文案推荐口径需求

日期：2026-03-23

## 目标

调整安装相关提示词和说明文案，让用户在 `codex` / `claude-code` 安装路径中看到更准确、更柔和的表达。

## 必须满足

- 把 “未安装 hook” 表述为当前作者仍在处理兼容性，因此暂未开放该安装面。
- 明确说明这不是用户安装失败，也不是用户少装了什么。
- 把未补的 MCP、`VCO_AI_PROVIDER_*`、相关本地 provider 配置表述为可选增强设置。
- 明确用户可以按需继续让 AI 帮忙补这些增强项，而不是把它们默认说成缺失告警。
- 安装提示整体改为推荐口吻，不使用容易让用户误判为异常或失败的警告口吻。

## 非目标

- 不改变安装命令本身。
- 不修改运行时逻辑或宿主支持边界。
- 不把可选增强项重新定义成默认必装项。
