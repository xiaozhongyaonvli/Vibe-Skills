# Public README Capability Snapshot Design

## Intent

在 README 首屏主标题与差异说明之间插入一个纯 Markdown 的“能力战报面板”，让用户在首屏第二视觉层立即看到项目的能力规模、治理密度与稳定执行特征。

## Chosen Approach

采用“能力战报面板”，而不是品牌海报式表达或伪终端式表达。

原因：

- 比品牌海报更适合承载真实能力信息
- 比伪终端更容易被普通用户理解
- 能与当前 README 的判断冲击型标题自然衔接

## Structure

展示区放在现有 `At A Glance` 位置，但从普通 bullets 升级为更有界面的能力快照：

1. 标题：`Capability Snapshot` / `能力快照`
2. 第一层：规模指标
3. 第二层：运行时与治理特征
4. 收束句：强调这不是静态目录，而是受管系统

## Content Rules

- 只写可验证事实，不写不可证的外部声誉
- 句长要短，字段感要强
- 保留部分控制台 / panel 质感，但不做极客化伪终端
- 与下方差异段形成“看数据 -> 看解释”的顺序

## Expected Effect

首屏浏览顺序变成：

- 标题判断
- 副标题解释
- 能力快照
- 差异说明

这样用户在最短时间内就能同时接收到：

- 这是什么
- 为什么强
- 为什么不是普通 skill 仓库
