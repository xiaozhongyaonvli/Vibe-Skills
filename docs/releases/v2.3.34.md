# VCO Release v2.3.34

- Date: 2026-03-13
- Commit(base): fac2756

## Highlights

- Upgraded the install messaging in the main README so “full-featured” now clearly means closure of the repo-governed payload, profile, and doctor surface, not a misleading promise that all host-managed dependencies are automatically provisioned.
- Added a dedicated cold-start onboarding track for first-time users and teams with three explicit paths: `minimum viable`, `recommended full-featured`, and `enterprise-governed`.
- Linked the new onboarding path from the main README, English README, docs index, and one-shot bootstrap guide so installation expectations, manual follow-up, and stop rules are visible from the first entry point.
- Cleaned the Chinese install section wording so the operator notes remain readable and consistent with the actual bootstrap behavior.

## Migration Notes

- No code or routing migration is required.
- Existing `v2.3.33` installs remain valid; this release improves operator guidance and onboarding clarity rather than changing runtime behavior.
- Teams that previously treated “install succeeded” as “ecosystem ready” should now align on the stricter readiness interpretation: repo-owned closure can be complete while host-managed surfaces still remain `manual_actions_pending`.
